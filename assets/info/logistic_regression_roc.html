ROC & Diagnostic Performance

This tool evaluates the diagnostic accuracy of a test or biomarker using the ROC curve. For a single analyte, no regression is involved — patients are ranked by their raw analyte value and the ROC curve is computed directly. When multiple analytes are entered, the tool fits a logistic regression model to combine them into a single predicted probability, which is then evaluated on its own ROC curve alongside each individual analyte.

When to use this tool

  • Evaluating a new biomarker's diagnostic accuracy (sensitivity, specificity, AUC)
  • Choosing a decision threshold for a clinical test
  • Comparing the discriminatory performance of two or more analytes
  • Combining multiple analytes to improve diagnostic accuracy beyond any single marker

ROC Curve

The Receiver Operating Characteristic curve plots sensitivity (TPR) on the y-axis against 1 − specificity (FPR) on the x-axis at every possible decision threshold. Each point on the curve represents one threshold — moving along the curve trades sensitivity for specificity.

  • AUC = 0.50 — the diagonal: coin-flip test, no diagnostic value
  • AUC = 0.70–0.80 — fair discrimination
  • AUC = 0.80–0.90 — good discrimination
  • AUC ≥ 0.90 — excellent discrimination
  • AUC = 1.00 — perfect discrimination (rare in real data)

AUC has an intuitive probabilistic interpretation: it equals the probability that a randomly chosen positive case scores higher than a randomly chosen negative case.

Cut-off selection

Youden index (J)

J = Sensitivity + Specificity − 1. The Youden cut-off maximises J, giving the best balance between sensitivity and specificity. It is the standard starting point but is not always the right clinical choice.

Clinical trade-off

The appropriate cut-off depends on the relative cost of false negatives vs false positives:

  • Screening test (rule out): minimise false negatives → choose a low threshold → high sensitivity, lower specificity. Example: D-dimer for DVT rule-out.
  • Confirmatory test (rule in): minimise false positives → choose a high threshold → high specificity, lower sensitivity. Example: troponin at a high percentile for AMI confirmation.

Performance metrics

Metric Formula Interpretation
Sensitivity TP / (TP + FN) Proportion of disease cases correctly detected
Specificity TN / (TN + FP) Proportion of non-disease cases correctly excluded
PPV TP / (TP + FP) Probability that a positive result is a true case — depends on prevalence
NPV TN / (TN + FN) Probability that a negative result truly excludes disease — depends on prevalence
LR+ Sens / (1 − Spec) How much a positive result increases the odds of disease. LR+ ≥ 10 = strong
LR− (1 − Sens) / Spec How much a negative result reduces the odds of disease. LR− ≤ 0.1 = strong

Logistic regression — combining analytes

When multiple analyte columns are present, the tool fits a logistic regression model:

log(p / 1−p) = β₀ + β₁x₁ + β₂x₂ + … + βₙxₙ

Each analyte is standardised before fitting to ensure numerical stability. The model outputs a predicted probability (0–1) for each patient, which is then used to build the combined ROC curve. The combined AUC is typically higher than any individual analyte because the model exploits partially independent information from each predictor.

Confidence interval on AUC

AUC 95% CIs are computed by non-parametric bootstrap (600 resamples). Overlapping CIs between two analytes suggest their AUC difference may not be statistically significant. For formal comparison, DeLong's test (not implemented here) should be used.

Accuracy & limitations

Accuracy & limitations: The logistic regression model is fit by gradient descent without regularisation — it may overfit with very small sample sizes (rule of thumb: ≥ 10 events per predictor). AUC estimated on the same data used to fit the model is optimistically biased; for valid model evaluation use a held-out test set or cross-validation. PPV and NPV depend heavily on prevalence — values shown here reflect the prevalence in the data you entered, which may not match your clinical population. This tool is intended for education and exploratory analysis, not regulatory submissions.
